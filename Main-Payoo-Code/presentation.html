Payoo

A responsive digital-wallet and personal-finance demonstration built with HTML, Tailwind/DaisyUI, and Vanilla JavaScript.

01 · Problem

Everyday financial tasks should feel simple.

Payoo brings common wallet tasks into one clear interface: move money, pay bills, manage savings, and understand account activity.

Clarity

Guided forms, validation, summaries, and receipts.

Convenience

Quick actions, saved data, recent contacts, and reminders.

Confidence

PIN checks, confirmation steps, Undo, and balance visibility.

02 · Objectives

Project goals

Responsive banking experience

Support mobile, tablet, and desktop layouts with the same user flow.

Practical JavaScript

Use validation, modular controllers, browser storage, events, and dynamic rendering.

Usable finance tools

Provide savings, budgeting, exchange conversion, transaction history, and QR tools.

Safe demo behavior

Keep financial actions understandable and reversible where appropriate.

03 · User flow

One consistent journey

Login→Dashboard→Select action→Validate details→Confirm→Updated balance & history

The same predictable pattern is used for cashout, send money, and bill payment.

04 · Core features

Digital wallet actions

Add Money

Validated deposits with draft support.

Cashout

Agent validation, PIN verification, confirmation summary.

Send Money

Contact lookup, review screen, recent and pinned contacts.

Pay Bill

Categories, saved bills, reminders, and confirmation.

Savings

Create goals, track progress, add money, and restore accidental deletions.

Bonus & Donate

Additional wallet-style interactions.

05 · Finance tools

More than transactions

Budget calculator

Plan budgets and save scenarios.

Currency tools

Convert currencies and view a currency database.

QR scanner

Camera/text scan interface with scan history.

Calendar

Organize dates and future planning.

Charts

Dashboard visualizes balance and activity data.

Notifications

Useful updates appear in the desktop shell.

06 · Transaction management

Transparent account activity

  • Search, filter, sort, and paginate transactions.
  • Export transaction data to CSV and use a print-friendly view.
  • Print a receipt for an individual transaction.
  • Remember the user’s filter and sorting preferences.

07 · UX & safety improvements

Designed to reduce mistakes

Confirm sensitive actions

Cashout and bill payments show a final summary before they run.

Undo non-financial deletes

Savings goals, saved bills, and reminders can be restored after deletion.

Session warning

Inactive sessions display a Continue Session prompt.

Data control

Profile explains local browser storage and provides Reset Demo Data.

08 · Accessibility & polish

Small details matter

  • `aria-live` support for toast messages.
  • Keyboard focus trapping inside dialogs.
  • Tooltips and labels for icon-only controls.
  • Keyboard shortcuts and clear focus styles.
  • Helpful empty states and formatted BDT amount previews.
  • Balance visibility control for privacy.

10 · Architecture

Organized for maintainability

HTML page→Page controller→Validation + DOM utilities→Mock BankService→Browser storage

Shared modules handle the desktop shell, UX behavior, charts, notifications, and cross-page enhancements.

11 · Testing

What was verified

Functional checks

Login, validation, balances, financial actions, savings, bill reminders, receipts, exports, and resets.

Interface checks

Desktop sidebar, tablet behavior, mobile layouts, empty states, keyboard actions, and dialogs.

Code checks

Updated JavaScript modules passed syntax validation.

Flow protection

Improvements were added around existing controllers rather than replacing the user journey.

12 · Limitations

Demo today, production tomorrow

  • Transactions are simulated, not connected to a real bank.
  • Browser storage is only suitable for a demo.
  • Production PINs and authentication require a secure server and database.
  • Exchange rates and fee values are illustrative.
